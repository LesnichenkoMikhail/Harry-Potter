# Гарри Поттер: анализ текстов

Shiny-приложение для анализа корпуса книг серии Harry Potter: эмоциональная динамика, ключевые слова, персонажи, сеть совместных упоминаний и классификация фрагмента по книге.

## Ссылка

https://tarasovaleksandr.shinyapps.io/harry-potter-main/

## Быстрый запуск после клонирования

```bash
git clone https://github.com/LesnichenkoMikhail/Harry-Potter.git
cd Harry-Potter
```

Установите зависимости из `DESCRIPTION`:

```r
deps <- trimws(unlist(strsplit(read.dcf("DESCRIPTION")[, "Imports"], ",")))
install.packages(deps)
```

Запустите приложение из корня репозитория:

```r
shiny::runApp()
```

Модель уже сохранена в папке `models/`, поэтому при обычном запуске ничего переобучать не нужно.

## Что есть в приложении

- Анализ тональности по главам с выбором лексикона Bing / AFINN.
- TF-IDF ключевых слов по каждой книге: столбчатый график или облако слов.
- Подсчет упоминаний основных персонажей.
- Сеть совместных упоминаний персонажей с фильтром минимальной силы связи.
- Классификатор: пользователь вводит фрагмент текста, приложение предсказывает наиболее похожую книгу.
- Таблица качества модели на тестовых главах.

## Как устроена модель

Исходный датасет содержит 95 085 коротких текстовых чанков. В разных книгах и главах разное количество чанков, поэтому модель обучается не на отдельных строках и не на целых главах, а на окнах по 100 последовательных чанков внутри главы.

Схема обучения:

- текст очищается;
- чанки объединяются в модельные документы по 100 чанков;
- главы делятся на `train`, `validation` и `test`;
- документы из одной главы не попадают в разные части выборки;
- используются TF-IDF признаки, длина текста, доля уникальных слов, частоты персонажей и характерных терминов;
- итоговая модель: weighted multinomial logistic regression (`glmnet`);
- финальная модель для приложения обучается на `train + validation`;
- `test` не используется при обучении и нужен только для итоговой оценки.

Модель не обучается при старте Shiny-приложения. При запуске загружаются готовые файлы:

- `models/classifier_fit.rds`
- `models/classifier_metrics.rds`

Разметка модельных документов сохранена в:

- `data/classifier_split_documents.csv`

## Переобучение модели

Переобучать модель для запуска приложения не нужно. Но если данные или признаки изменились, модель можно пересобрать:

```bash
Rscript scripts/train_classifier.R
```

После переобучения обновятся:

- `models/classifier_fit.rds`
- `models/classifier_metrics.rds`
- `data/classifier_split_documents.csv`

Эти файлы хранятся в репозитории намеренно: другой человек после клонирования может сразу открыть приложение без долгого обучения модели.
